# 工作区板块化布局设计

> 项目：文澜排版 VellumStyle  
> 日期：2026-07-12  
> 范围：重构桌面工作区的视觉层级和编辑器/预览布局，不改变文档、渲染、发布与样式编辑业务逻辑。

## 背景

当前工作区将文档管理、大纲、编辑器和预览连续铺满，核心区域主要依靠细分隔线区分。功能完整，但编辑器与预览缺少清晰的板块边界，顶部全局栏同时承载导航、编辑和文档操作，操作归属不够明确。

本设计借鉴 R-Markdown 的“框架底色 + 独立内容表面”思路，但保留 VellumStyle 的双独立侧栏、本地文档体系、预览模式和浮动样式检查器。

## 目标

- 让编辑器和预览成为工作区中清晰、稳定的两个核心板块。
- 将 Markdown 语法操作归还给编辑器，减少全局顶栏的信息拥挤。
- 保留文档管理与大纲两个可独立开关、可同时展开的抽屉式侧栏。
- 允许用户调整编辑器与预览的空间分配，并记住选择。
- 保持当前预览模式、双向同步滚动、样式编辑和状态栏工作流不变。

## 非目标

本轮不包含：

- 深色模式。
- 移动端专用布局。
- 面板自由停靠、换位或多套工作区预设。
- 给预览区增加标题栏。
- 移动状态栏中的预览模式切换器。
- 重构 Markdown 渲染、文档存储、同步、发布或主题模型。

## 最终布局

```text
┌────────────────────────── 全局顶栏 ──────────────────────────┐
├─────────────────────────────────────────────────────────────┤
│ 工作区框架底色                                              │
│ ┌──────────┐ ┌────────┐  ┌─ 编辑器局部工具栏 ───────┐ ║ ┌──────────┐ │
│ │ 文档管理  │ │ 大纲    │  ├─────────────────────────┤ ║ │ 预览      │ │
│ │ 独立抽屉  │ │ 独立抽屉│  │ Markdown 编辑内容       │ ║ │          │ │
│ │ 可折叠    │ │ 可折叠  │  │                         │ ║ │          │ │
│ └──────────┘ └────────┘  └─────────────────────────┘ ║ └──────────┘ │
│                                                     ↑ 可拖动       │
├────────────────────────── 状态栏 ────────────────────────────┤
│                              流式 / 微信桌面 / 手机预览模式   │
└─────────────────────────────────────────────────────────────┘
```

文档管理和大纲都是可选区域。它们可以全部收起、单独展开或同时展开；同时展开时固定按“文档管理 → 大纲 → 编辑器 → 预览”的顺序排列。

## 关键设计决策

### 1. 工作区框架与核心板块

- 主体区域使用独立的工作区框架底色，与面板表面形成明度差。
- 桌面工作区使用约 `10px` 的外围留白和约 `8px` 的核心面板间距。
- 编辑器与预览使用独立表面、轻边框和约 `10px` 圆角。
- 默认不使用明显投影；活动状态优先通过边框或细强调线表达，避免仪表盘式卡片堆叠。
- 顶部全局栏和底部状态栏继续贴合窗口边缘，不参与卡片化。

### 2. 双独立抽屉

- 文档管理和大纲沿用顶部两个独立开关及各自的激活状态。
- 两个抽屉互不排斥，可以同时展开。
- 抽屉展开后参与横向布局并推动核心工作区，不覆盖编辑内容。
- 文档管理保留现有拖动调宽能力；大纲默认保持约 `220px` 的紧凑宽度。
- 抽屉保留侧栏语义和次级底色，不做成与编辑器、预览同等级的悬浮卡片。
- 展开/收起使用短促、克制的位移与透明度过渡，并遵循 `prefers-reduced-motion`。
- 不因窗口变窄自动关闭任一抽屉；在空间不足时，核心分栏的宽度约束应根据实时可用空间收敛，保持现有“用户显式控制侧栏”的原则。

### 3. 仅编辑器增加局部标题栏

- 只有编辑器板块增加局部标题栏；预览板块不增加标题栏。
- 标题栏高度保持紧凑，主要承担 Markdown 语法工具的归属，不增加醒目的大标题。
- 当前 `SyntaxToolbar` 从全局顶栏左侧移动到编辑器局部标题栏。
- 全局顶栏左侧继续保留文档管理与大纲开关。
- 导入、主题、上传、发布、复制、设置等整篇文档或应用级操作继续由 `MainToolbar` 承担。
- 编辑器标题栏和正文属于同一个圆角面板，二者使用轻分隔线区分。

### 4. 编辑器/预览宽度调节

- 编辑器和预览之间增加可见但克制的拖动柄。
- 拖动改变的是两个外层板块的空间比例，不替代预览内部的流式、微信桌面和手机内容宽度模式。
- 分隔柄使用 Pointer Events，避免只支持鼠标。
- 分隔柄可聚焦，并支持方向键小步调整；使用修饰键时可大步调整。
- 双击分隔柄恢复默认比例。
- 拖动期间禁用文本选择，并在结束或取消时可靠清理捕获状态。
- 比例根据当前核心工作区可用宽度进行约束，避免任一板块被拖到不可操作。
- 用户设置的比例写入现有 Zustand 持久化状态；无值或值无效时使用默认比例。

### 5. 预览与样式面板保持现状

- 预览板块不增加局部标题栏，保持内容区域干净。
- `PreviewModeToggle` 继续位于底部状态栏。
- 流式、微信桌面和手机模式继续只控制预览文章容器宽度。
- 编辑器与预览的双向同步滚动逻辑保持不变。
- 点击预览元素后出现的 `StylePanel` 继续作为右侧浮动 Inspector，不占据固定工作区列。
- StylePanel 的出现不改变编辑器/预览已保存的分栏比例。

## 组件与状态边界

### `App.tsx`

负责组合全局顶栏、双抽屉、工作区框架、核心分栏和状态栏。它只协调打开状态与布局，不承担拖动比例计算细节。

### 编辑器面板壳

新增一个聚焦于编辑器区域的面板壳，负责：

- 圆角、边框和表面样式。
- 承载编辑器局部标题栏。
- 承载现有 `SyntaxToolbar` 和 `MarkdownEditor`。
- 表达编辑器焦点状态。

不为预览强行复用带标题栏的壳；共享视觉令牌通过 CSS/Tailwind 类实现，避免制造条件复杂的万能组件。

### 分栏布局模块

将比例约束、像素换算、键盘步进和默认值恢复设计为纯函数，UI 组件只负责指针事件与渲染。这样可以独立测试窗口宽度、双抽屉展开和极端持久化值等边界。

### Zustand 状态

新增一个持久化的工作区比例字段和设置动作。文档管理、大纲的展开状态继续保持运行期状态，不改变现有启动行为。

## 数据流

1. 应用从 Zustand 读取文档抽屉、大纲抽屉的打开状态以及持久化分栏比例。
2. 两个抽屉按各自状态参与或退出主布局。
3. 核心工作区测量当前可用宽度，并将比例转换为编辑器和预览的实际宽度。
4. 用户拖动或键盘调整分隔柄时，布局即时更新。
5. 调整结束后，将归一化比例写入 Zustand 持久化状态。
6. 预览模式仍由现有 `previewMode` 状态独立控制，不与外层比例互相覆盖。

## 可访问性与交互

- 文档和大纲开关继续使用 `aria-pressed` 表达打开状态。
- 分隔柄使用可聚焦的 separator 语义，提供当前值、最小值、最大值和操作说明。
- 键盘用户可以用方向键调整比例，并能恢复默认值。
- 面板焦点不能依赖颜色作为唯一提示；边框变化需要保持足够可见。
- 所有新增动画支持减少动态效果偏好。
- 拖动柄的实际命中区域不小于约 `8px`，视觉线条可以更细。

## 边界与降级

- 持久化比例缺失、非有限数或越界：回退到默认比例。
- 工作区宽度变化：重新约束实际宽度，但不擅自关闭抽屉。
- 两个抽屉同时展开：仍保留两者，核心分栏按剩余空间重新计算。
- 拖动中窗口失焦、指针取消或组件卸载：结束拖动并恢复文本选择。
- 预览正在加载或显示骨架屏：分栏宽度仍可调。
- StylePanel 打开：不重写或重算持久化比例。

## 测试策略

### 单元测试

- 比例转像素和像素转比例。
- 默认比例恢复。
- 非法持久化值回退。
- 在不同可用宽度下执行最小/最大约束。
- 键盘小步、大步调整与恢复默认值。
- 文档管理和大纲四种组合状态的布局顺序。

### 组件测试

- SyntaxToolbar 只出现在编辑器标题栏，不再出现在全局顶栏。
- 预览区没有新增标题栏。
- PreviewModeToggle 仍位于状态栏。
- 分隔柄具有正确语义、键盘行为和双击恢复行为。
- 两个抽屉可以同时打开，且顺序稳定。

### 回归验证

- 编辑、预览、双向同步滚动正常。
- 文档树拖动调宽正常。
- 大纲跳转与活动标题高亮正常。
- 预览三种模式正常。
- 点击预览元素可正常打开和关闭 StylePanel。
- 导入、上传、发布、复制、主题与设置入口不受影响。
- `npm test` 与 `npm run build` 通过。

## 验收标准

- 默认窗口下，编辑器和预览能明显识别为两个独立核心板块。
- 只有编辑器具有局部标题栏，且 SyntaxToolbar 位于其中。
- 文档管理与大纲可以分别开关，也可以同时展开。
- 两个抽屉同时打开时不互相覆盖，顺序符合设计。
- 编辑器/预览比例可通过指针和键盘调整，并在重启后恢复。
- 预览模式切换器仍在状态栏，功能与视觉模式不回退。
- StylePanel 保持浮动 Inspector 行为。
- 未引入暗色模式、移动端专用布局或自由停靠系统。
