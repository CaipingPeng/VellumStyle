# 属性侧栏紧凑表单设计

**日期：** 2026-07-14

## 目标

只优化渲染区元素被选中后，右侧属性面板的表单密度与排列方式。采用已确认的“A：平衡网格”方案，让同类短参数在一行内并排编辑。

## 范围

- 修改 `src/components/StylePanel/StylePanel.tsx` 的前端表单布局。
- 修改 `src/components/StylePanel/controls.tsx` 的紧凑控件样式。
- 允许在上述文件内部增加少量只用于展示的辅助函数或常量。
- 不新增独立布局模块。
- 不修改 store、主题模型、样式更新路径、保存流程或 `src-tauri`。

## 布局规则

- 字号、行高、字间距等短数值字段使用三列网格。
- 文字颜色、字重、对齐等较宽控件使用更宽跨度。
- `marginPadding` 中的四向外边距和内边距按上、右、下、左分成四列。
- 其他嵌套短字段使用自适应紧凑网格。
- 颜色、复合值、长文本与高级 CSS 保留足够宽度；未知字段安全回退为整行。
- 保持原有字段 path，输入变化仍调用现有 `handleStyleChange(path, value)`。

## 交互和兼容性

- 控件行为、单位解析、颜色选择、临时修改和永久保存行为不变。
- 面板宽度和开关动画不变。
- 只改变视觉分组、间距、标签尺寸和 Grid 占位。
- 在小视口下允许网格退化，避免输入框不可操作。
