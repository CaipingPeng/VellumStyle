# 渲染区图片右键操作设计

**日期：** 2026-07-14
**状态：** 已获用户确认

## 目标

为文章渲染区中的图片增加专用右键菜单。用户可将所指图片直接复制到系统剪贴板，或通过系统“另存为”对话框选择路径和文件名后保存图片。

## 范围

### 包含

- 仅对渲染区正文内的 `<img>` 元素显示图片右键菜单。
- 菜单包含“拷贝图片”和“将图片另存为”两个选项。
- 剪贴板内容统一为 PNG 位图；GIF 动画和 SVG 等格式允许转换为 PNG 静态图。
- 另存为尽量保留原始图片字节、格式和文件名。
- 支持当前预览中的微信代理图片、普通 HTTP(S) 图片及 data URL 图片。
- 成功与失败使用现有 Toast 反馈；取消另存为不显示错误。
- 增加前端单元/交互测试、Rust 图片处理测试以及完整构建验证。

### 不包含

- 编辑图片内容、裁剪、缩放后再保存或复制。
- 复制图片的 Markdown、URL、HTML 或替代文本。
- 批量复制或批量保存图片。
- 保留 GIF 动画到剪贴板。
- 给非图片区域增加自定义右键菜单。

## 交互设计

用户在文章渲染区正文图片上右击时，阻止 WebView 默认菜单并在指针附近显示应用内菜单，顺序固定为：

1. 拷贝图片
2. 将图片另存为

菜单遵循应用现有颜色变量，支持明暗外观，并限制在当前视口内，避免从右侧或底部溢出。以下操作关闭菜单：

- 选择任一菜单项；
- 点击菜单外部；
- 按 `Escape`；
- 滚动渲染区；
- 窗口失焦；
- 预览内容重新渲染。

右击非图片区域时不显示该菜单，也不阻止默认行为。右键操作不改变图片现有的悬停、选中、样式面板和四角拖动缩放逻辑。

操作反馈：

- 复制成功：Toast 显示“图片已复制”。
- 保存成功：Toast 显示“图片已保存”。
- 用户取消另存为：静默关闭，不显示 Toast。
- 获取、解码、复制或写盘失败：Toast 显示“图片复制失败：…”或“图片保存失败：…”，且不得误报成功。

## 架构与职责

采用“React 交互 + Tauri 原生图片处理”的混合方案。

### 前端预览交互

`src/components/Preview/Preview.tsx` 负责：

- 通过事件委托识别 `#article` 内被右击的真实 `<img>`；
- 保存目标图片地址与菜单坐标；
- 渲染、定位和关闭菜单；
- 调用独立图片操作工具；
- 根据结果显示 Toast。

菜单拆为小型展示组件，业务与图片字节处理不写入菜单 JSX。目标图片以右击瞬间捕获的地址为准，后续悬停或选中变化不能切换操作对象。

### 前端图片操作工具

新增聚焦于图片动作的工具模块，负责：

- 将微信预览代理 URL 还原为原始 URL；
- 调用 Tauri 后端取得原始图片元数据和字节；
- 调用原生剪贴板能力写入 PNG；
- 调用 `@tauri-apps/plugin-dialog` 打开保存对话框；
- 使用现有或共用的字节写盘命令保存用户选择的路径；
- 在 Web 调试模式提供明确错误，而不是假装复制或保存成功。

### Tauri 图片后端

新增独立 Rust 模块并注册命令，负责：

- 安全解析受支持的图片源；
- 对 HTTP(S) 图片使用原生网络请求，避免浏览器 CORS 限制；
- 解码 data URL；
- 校验响应确实是受支持的图片，并限制可接受的响应体大小；
- 返回原始字节、可靠 MIME 类型、建议文件名和扩展名；
- 复制时将栅格图片、GIF 首帧或 SVG 渲染结果统一转换为 PNG；
- 通过 Tauri 原生剪贴板能力写入系统图片剪贴板。

不得把任意响应无校验地当作图片，也不得因 URL 文件扩展名与实际内容冲突而写入错误格式。远程请求设置超时和最大体积，错误转换为可理解的中文消息。

## 数据流

### 拷贝图片

1. `contextmenu` 事件捕获目标 `<img>` 和地址。
2. 用户点击“拷贝图片”，菜单立即关闭。
3. 前端还原微信代理地址并调用 Tauri 复制命令。
4. 后端读取并校验图片，解码后编码成 PNG。
5. 后端把 PNG/位图写入系统剪贴板。
6. 前端仅在命令成功返回后显示成功 Toast。

### 将图片另存为

1. `contextmenu` 事件捕获目标 `<img>` 和地址。
2. 用户点击“将图片另存为”，菜单立即关闭。
3. 前端请求后端取得原始图片字节及文件元数据。
4. 前端以建议文件名和识别出的格式打开系统保存对话框。
5. 用户取消时结束流程；用户确认时将原始字节写入所选路径。
6. 写盘成功后显示成功 Toast。

先取元数据再打开对话框，保证默认文件名和过滤器与实际格式一致。保存不使用预览元素经过 CSS 缩放后的像素，也不把 GIF 强制转成 PNG。

## 文件名与格式规则

- MIME 类型与文件签名优先于 URL 后缀。
- URL 路径中存在安全、非空文件名时复用其主体名称。
- 文件名中的路径分隔符和 Windows 非法字符替换为安全字符。
- URL 无文件名时使用 `image`。
- 扩展名按检测出的实际格式补全或纠正。
- 无法可靠识别原始格式时拒绝保存并显示错误，不写入扩展名与内容不匹配的文件。

## 安全与资源限制

- 后端只接受 `http:`、`https:` 和 `data:image/...`；拒绝 `file:`、UNC 路径及其他任意协议。
- HTTP(S) 请求设置连接/总超时并限制重定向数量。
- 下载过程限制最大图片字节数，避免无界内存占用。
- data URL 同样执行大小限制和格式校验。
- SVG 只作为图片解码/渲染输入，不执行脚本或外部资源。
- 前端错误消息不暴露密钥、完整响应体或内部堆栈。

## 测试设计

### 前端纯函数与交互

- 仅 `#article` 内的图片可成为菜单目标。
- 微信代理 URL 能正确还原为原始地址。
- 菜单坐标在视口右侧、底部会被钳制。
- 菜单项顺序和文案正确。
- 点击外部、`Escape`、滚动和重新渲染会关闭菜单。
- 点击复制/保存时使用右击瞬间捕获的图片地址。
- 复制、保存成功和失败分别触发正确 Toast；取消保存保持静默。
- 现有图片缩放和样式选择测试继续通过。

### Rust

- PNG、JPEG、GIF 输入可识别实际格式；复制输出为有效 PNG。
- GIF 复制使用静态帧。
- SVG 输入可安全渲染为 PNG。
- data URL 可解码，畸形或超限输入被拒绝。
- HTTP 非成功状态、非图片响应、超时和超限响应被拒绝。
- 文件名清理、缺省名称和扩展名纠正规则正确。
- 不受支持的协议被拒绝。

### 完整验证

- `npm test`
- `npm run build`
- `cargo test --manifest-path src-tauri/Cargo.toml`
- Windows Tauri 手工冒烟：分别验证 PNG/JPEG/GIF、微信代理图片的复制和另存为；确认可粘贴到画图或聊天输入框，保存文件可正常打开。

## 验收标准

- 渲染区任意正文图片右击时出现且只出现两个指定菜单项。
- “拷贝图片”成功后，系统剪贴板中是对应图片的 PNG 位图，可粘贴到支持图片的 Windows 应用。
- GIF/SVG 复制为静态 PNG 符合用户确认的格式策略。
- “将图片另存为”打开系统保存对话框，允许用户选择目录并填写名称。
- 保存内容对应原图，尽量保留原格式；文件扩展名与实际字节一致。
- 用户取消保存不会创建文件，也不会显示失败提示。
- 网络、格式、解码、剪贴板或写盘失败均有明确反馈且不误报成功。
- 原有预览点击选样式、悬停高亮、滚动同步和图片拖动缩放行为不回归。
