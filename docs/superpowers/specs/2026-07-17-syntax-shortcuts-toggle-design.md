# 编辑器语法快捷键与可逆格式设计

日期：2026-07-17

## 目标

为编辑器现有 Markdown 语法按钮补充键盘快捷键，并让按钮点击与快捷键复用同一套语法命令。可逆格式支持再次执行取消，降低键盘编辑成本，同时避免按钮和快捷键出现行为差异。

## 已确认范围

- 覆盖现有语法动作：加粗、斜体、删除线、行内代码、链接、H1–H4、有序列表、无序列表、引用、代码块、分割线。
- 快捷键仅在 CodeMirror 编辑器获得焦点时生效。
- 支持 Windows、Linux 和 macOS；除分割线外，使用 Typora 官方默认快捷键。
- 加粗、斜体、删除线、行内代码、标题、列表、引用和代码块支持添加/取消切换。
- 链接与分割线保持插入型语义。
- 不改变工具栏布局、图标或视觉样式。

## 快捷键映射

| 语法动作 | Windows/Linux | macOS |
|---|---|---|
| 加粗 | `Ctrl+B` | `Cmd+B` |
| 斜体 | `Ctrl+I` | `Cmd+I` |
| 删除线 | `Alt+Shift+5` | `Control+Shift+反引号` |
| 行内代码 | `Ctrl+Shift+反引号` | `Cmd+Shift+反引号` |
| 链接 | `Ctrl+K` | `Cmd+K` |
| H1 | `Ctrl+1` | `Cmd+1` |
| H2 | `Ctrl+2` | `Cmd+2` |
| H3 | `Ctrl+3` | `Cmd+3` |
| H4 | `Ctrl+4` | `Cmd+4` |
| 有序列表 | `Ctrl+Shift+[` | `Cmd+Option+O` |
| 无序列表 | `Ctrl+Shift+]` | `Cmd+Option+U` |
| 引用 | `Ctrl+Shift+Q` | `Cmd+Option+Q` |
| 代码块 | `Ctrl+Shift+K` | `Cmd+Option+C` |
| 分割线 | `Ctrl+Shift+H` | `Cmd+Shift+H` |

Typora 并非在所有动作上简单地将 Windows 的 `Ctrl` 换成 macOS 的 `Cmd`。实现需为平台分别配置键位。分割线没有 Typora 官方默认快捷键，本项目使用助记键 `Mod+Shift+H`（Horizontal Rule）。

参考：[Typora 官方快捷键文档](https://support.typora.io/Shortcut-Keys/ "Shortcut Keys - Typora Support")。

## 统一命令架构

### 语法动作

定义有限的动作标识：

```ts
type SyntaxAction =
  | "bold"
  | "italic"
  | "strikethrough"
  | "inlineCode"
  | "link"
  | "heading1"
  | "heading2"
  | "heading3"
  | "heading4"
  | "orderedList"
  | "unorderedList"
  | "blockquote"
  | "codeBlock"
  | "horizontalRule";
```

编辑器句柄新增统一入口：

```ts
runSyntaxAction(action: SyntaxAction): void
```

工具栏按钮和 CodeMirror 快捷键都调用同一个动作执行器：

```text
点击工具栏按钮 ─┐
               ├─→ runSyntaxAction(action)
键盘快捷键 ────┘       ↓
                 判断添加或取消语法
                       ↓
                 CodeMirror transaction
                       ↓
                 现有 onChange 与预览更新
```

### 文件职责

- `src/components/Editor/syntaxActions.ts`
  - 定义 `SyntaxAction`。
  - 定义 Typora 平台快捷键映射。
  - 生成 CodeMirror keymap。
  - 将动作分派给统一命令执行器。
- `src/components/Editor/editing.ts`
  - 承载可测试的文本变换、范围判断和选区映射逻辑。
- `src/components/Editor/MarkdownEditor.tsx`
  - 注册语法 keymap。
  - 结合 CodeMirror Markdown 语法树识别当前上下文。
  - 暴露 `runSyntaxAction` 给工具栏。
- `src/components/Toolbar/SyntaxToolbar.tsx`
  - 每个按钮只提交对应的 `SyntaxAction`，不再分别决定使用包裹、行前缀或代码块方法。

每次命令使用单个 CodeMirror transaction 完成，使一次撤销能够完整恢复该操作。编辑器尚未初始化或不可编辑时，命令安全退出。

## 语法切换规则

### 行内格式

适用于加粗、斜体、删除线和行内代码。

1. 判断当前选区或光标是否包含在对应的完整语法节点中。
2. 若存在对应节点，移除节点的首尾标记，保留内部文字。
3. 若不存在，包裹选区；无选区时插入并选中现有占位文字。

使用 CodeMirror Markdown 语法树识别：

- `StrongEmphasis`
- `Emphasis`
- `Strikethrough`
- `InlineCode`

语法树识别可避免仅靠正则时对转义字符、星号和嵌套格式的误判。嵌套格式独立切换：执行哪一种格式，就只添加或移除对应层级。选区跨越多个独立的同类节点时，不猜测批量取消，而是将整个选区作为新的目标范围。

切换后继续选中内部文字。光标位于格式内部但没有选区时，取消外层格式并维持相对于内部文字的位置。

### 标题

- 正文执行 Hn：添加对应的 `#` 标记。
- Hn 再执行相同 Hn：移除标题标记，恢复正文。
- 一个标题等级执行另一个等级：直接替换等级，不叠加标记。
- 多行选区对涉及的每一行应用同一目标等级。

### 列表

- 普通行执行有序或无序列表：添加目标列表标记。
- 已是同类列表：移除列表标记。
- 已是另一类列表：直接转换列表类型。
- 保留正文前的缩进。
- 多行选区逐行处理，空行不强制添加列表标记。
- 有序列表沿用现有写法，每项使用 `1. `，交由 Markdown 渲染器编号。

### 引用

- 普通行添加一层 `> `。
- 已引用行移除一层引用。
- 多层引用每次只增减一层。
- 多行选区只有在所有非空行都已有引用标记时统一取消；否则为所有非空行统一添加一层。

### 代码块

使用 CodeMirror 语法树中的 `FencedCode` 识别围栏代码块。

以下情况均取消所属代码块：

- 完整选中代码块，包括围栏；
- 光标位于代码内容、语言标识或首尾围栏行；
- 选区只覆盖代码块内部的一部分，但整个选区包含在同一个代码块中。

取消时移除首尾围栏和语言标识，保留代码正文，并将光标或选区映射到保留内容的等效位置。反引号围栏和波浪线围栏都可识别；新建代码块继续使用三个反引号。

以下情况不执行猜测性取消，而按添加代码块处理：

- 选区跨越多个代码块；
- 只有单侧围栏；
- 围栏不完整；
- 没有找到唯一、完整且包含目标范围的 `FencedCode` 节点。

### 插入型语法

- 链接继续插入 `[文字](链接地址)`，不把再次执行解释为删除链接，避免静默丢弃 URL。
- 分割线继续插入 `\n---\n`，不做切换。
- 图片属于上传与内容插入流程，不纳入语法快捷键。

## 选区和光标映射

- 行内格式添加或取消后，选区继续覆盖内部文字。
- 行级格式有选区时，映射原选区的方向和涉及行，确保立即再次执行能取消；无选区时，光标保留在当前行的等效正文位置。
- 代码块添加或取消后，光标/选区尽可能保持相对于代码正文的位置。
- 所有位置变换由同一 transaction 的 changes 映射完成，避免先改文档再手工猜测最终偏移。

## 快捷键注册

- 使用 CodeMirror `keymap.of(...)` 注册。
- 每条绑定配置独立的 `win`、`linux` 和 `mac` 键位。
- 使用 `Prec.highest(...)`，使编辑器聚焦时语法命令优先于 WebView 默认行为。
- 成功处理后返回 `true`，阻止 WebView 继续消费按键。
- 保留既有撤销、重做和 `Ctrl+H` 本地化查找功能。
- 快捷键不会注册到 `window` 或 `document`，因此不会干扰设置、发布、搜索输入框和其他应用区域。

## 错误处理与安全边界

- 找不到完整对应语法节点时，不执行破坏性取消。
- 不完整加粗标记、孤立反引号、单侧代码围栏等情况按添加语法处理。
- 对嵌套格式只操作目标节点，不清理无关语法。
- 编辑器未初始化或只读时直接返回。
- 命令不直接调用 `onChange`，继续由 CodeMirror transaction 驱动现有数据流。

## 测试策略

### 快捷键映射

- 验证 14 个语法动作都有快捷键。
- 逐项验证 Windows、Linux 和 macOS 映射。
- 验证分割线使用项目自定义的 `Mod+Shift+H`。

### 行内格式

- 有选区添加并再次取消。
- 无选区插入占位内容并再次取消。
- 光标位于语法内部时取消。
- 嵌套格式只取消目标层。
- 转义字符和不完整语法不被误删。

### 行级格式

- 标题添加、同级取消和跨级转换。
- 有序与无序列表互转及取消。
- 引用单层增减及多行统一判断。
- 缩进、选区范围和选区方向保持。

### 代码块

- 完整选中取消。
- 光标位于代码正文、语言标识和围栏行时取消。
- 局部选区位于同一个代码块时取消。
- 反引号与波浪线围栏取消。
- 跨多个代码块或不完整围栏不误取消。
- 光标和选区正确映射。

### 统一入口与回归

- 工具栏按钮与快捷键提交相同 `SyntaxAction`。
- 每次切换只产生一个可撤销步骤。
- 链接与分割线保持插入型行为。
- 既有编辑器输入、中文输入法、搜索、上传图片、同步预览不回归。
- 运行完整测试套件及 TypeScript/Vite 构建，并在桌面应用中手动验证 WebView 不消费快捷键。

## 验收标准

1. 编辑器聚焦时，所有确认的 Typora 快捷键执行对应语法动作；其他区域不响应。
2. 工具栏与快捷键对同一动作产生完全一致的结果。
3. 可逆格式支持选区或上下文感知取消，不损坏嵌套的其他格式。
4. 代码块支持完整选中或定位在块内时取消，并保留代码正文。
5. 标题、列表和引用支持添加、取消及合理转换。
6. 链接和分割线继续保持插入语义。
7. 单次撤销完整恢复一次语法操作。
8. 自动化测试和生产构建通过。
