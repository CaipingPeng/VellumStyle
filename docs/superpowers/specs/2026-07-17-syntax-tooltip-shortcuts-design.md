# 语法按钮平台自适应快捷键提示设计

日期：2026-07-17

## 目标

在不改变语法工具栏布局、图标和视觉样式的前提下，让鼠标悬停到语法图标时，原生提示同时显示语法含义和当前操作系统对应的快捷键。

示例：

- Windows/Linux：`加粗 (Ctrl+B)`
- macOS：`加粗 (⌘B)`

## 已确认范围

- 覆盖现有 14 个 `SyntaxAction`：加粗、斜体、删除线、行内代码、链接、H1–H4、有序列表、无序列表、引用、代码块和分割线。
- 提示按当前操作系统自适应，不同时罗列多个平台。
- 保持现有浏览器原生 `title` hover 提示，不引入自定义 Tooltip。
- 快捷键提示与 `SYNTAX_SHORTCUTS` 共用同一数据源，避免显示内容与实际绑定不一致。
- 标题图标是 H1–H4 的统一入口，合并显示为 `标题 (Ctrl+1–4)` 或 `标题 (⌘1–4)`。
- 上传、撤销和重做不属于 14 个语法动作，本次不调整。

## 方案选择

### 方案 A：从快捷键注册表生成原生 title（采用）

优点：不改变 UI；改动集中；快捷键绑定和展示共用数据源；易于测试。

### 方案 B：在工具栏手写各平台提示

优点是实现直接，但会重复维护快捷键，后续容易与真实绑定产生偏差，因此不采用。

### 方案 C：新增自定义 Tooltip

可以提供键帽样式，但会引入额外组件、动画和无障碍行为，也会改变现有视觉表现，超出当前需求，因此不采用。

## 架构设计

### 快捷键展示函数

在 `src/components/Editor/syntaxActions.ts` 中增加纯函数，负责：

1. 将运行环境归类为 `win`、`linux` 或 `mac`。
2. 根据 `SyntaxAction` 从 `SYNTAX_SHORTCUTS` 读取该平台的真实绑定。
3. 将 CodeMirror 键位记法转换成面向用户的显示文本。

平台识别允许显式传入平台值，便于单元测试；工具栏运行时使用浏览器平台信息。

### 显示格式

- Windows/Linux 使用常见文字修饰键：`Ctrl`、`Shift`、`Alt`。
- macOS 使用系统常见符号：`⌘`、`⌃`、`⇧`、`⌥`。
- 字母显示为大写，数字和括号等符号保持原样。
- 普通按钮格式为：`语法含义 (快捷键)`。
- 标题按钮将 H1–H4 合并成范围：
  - Windows/Linux：`标题 (Ctrl+1–4)`
  - macOS：`标题 (⌘1–4)`

### 工具栏接入

`src/components/Toolbar/SyntaxToolbar.tsx` 只负责把动作和中文含义交给展示函数，不复制快捷键字符串。按钮点击行为继续调用 `runSyntaxAction(action)`，本次改动不会触碰语法命令逻辑。

## 数据流

```text
SYNTAX_SHORTCUTS
   ├─ CodeMirror keymap：注册实际快捷键
   └─ 快捷键显示函数：生成当前平台标签
                         ↓
              SyntaxToolbar 的 title
```

## 边界与降级

- macOS 平台识别优先匹配 `Mac`、`iPhone`、`iPad`、`iPod`。
- Windows 匹配 `Win`。
- 其他或缺失的平台信息按 Linux/非 macOS 的 `Ctrl` 方案显示，保证始终有可读提示。
- 找不到动作配置时不静默猜测；类型和测试应保证 14 个动作完整覆盖。

## 测试策略

1. `syntaxActions.test.ts`：验证同一动作在 Windows、Linux、macOS 下生成正确显示文本，并覆盖复杂组合键。
2. `SyntaxToolbar.test.tsx`：分别注入平台环境，验证每个语法按钮的 `title`；验证标题范围提示。
3. 先写失败测试并确认因提示缺失而失败，再写最小实现。
4. 运行语法相关测试、全量 `npm test`、生产构建 `npm run build` 和 `git diff --check`。

## 验收标准

- 鼠标悬停任一语法图标时，同时看到语法含义和快捷键。
- Windows/Linux 与 macOS 显示各自真实绑定。
- 提示与快捷键注册表保持一致。
- 标题入口清楚表达 H1–H4 的快捷键范围。
- 工具栏视觉和点击行为不变。
