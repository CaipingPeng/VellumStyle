# 应用暗色模式设计

**日期：** 2026-07-12  
**状态：** 已确认

## 目标

为 VellumStyle 增加只包含亮色与暗色两种状态的应用外观模式。用户可在底部状态栏最右侧通过单个图标即时切换，选择持久化到本地；应用外壳、工作区、编辑器及各类浮层同步换色，但文章预览与导出结果继续完全由 Markdown 主题控制。

## 交互

- 状态栏最右侧新增无文字图标按钮，与预览宽度控件之间保留分隔线。
- 当前为亮色时显示 `Sun`，提示“切换到暗色模式”；当前为暗色时显示 `Moon`，提示“切换到亮色模式”。
- 点击后立即在两种模式间切换，不提供系统模式、下拉菜单或额外设置项。
- 按钮有可访问名称、键盘焦点样式，并通过 `aria-pressed` 表达暗色是否开启。

## 状态与持久化

新增独立的 `AppearanceMode = "light" | "dark"` 模型，默认亮色。Zustand store 持有 `appearanceMode` 和 `toggleAppearanceMode`，并将模式纳入现有 `vellumstyle` 持久化数据。恢复旧数据或无效值时统一回退亮色。

应用把状态同步到根元素的 `data-appearance` 属性，并同步 `color-scheme`，使原生控件与滚动条获得正确外观。入口在 React 挂载前从现有持久化数据读取一次模式，避免启动时先亮后暗；React 运行期继续以 store 为唯一状态源。

## 视觉架构

沿用现有 CSS 变量体系：亮色变量保持不变，`:root[data-appearance="dark"]` 覆盖应用语义变量，包括背景、工作区表面、边框、文字、状态色、阴影和焦点环。组件优先继续使用 `bg-bg`、`text-text` 等语义类，遗留的硬编码浅色应用表面改为变量或增加暗色覆盖。

暗色不是纯黑：框架、板块和浮层使用分层的深灰蓝，保证已确认的“明显板块”结构仍清晰；主色维持靛蓝体系，但提高暗色环境中的可读性。

## 编辑器

CodeMirror 外观跟随应用模式。通过独立 `Compartment` 重配置主题与 `dark` facet，而不是重建 EditorState/EditorView，因此切换不会改变正文、选区、输入法组合态、撤销历史或滚动位置。编辑器背景、正文、光标、选区、搜索面板均使用应用语义变量。

## 预览与导出隔离

应用暗色只影响预览板块的外壳和定宽模式外围。实际文章画布继续保持白色，注入到 `#article` 的 Markdown 主题 CSS 和代码主题 CSS不随应用模式变化。主题缩略图同样保留真实白色文章底色；复制、导出、发布链路不读取 `appearanceMode`，确保输出不被污染。

## 覆盖范围

- 顶部工具栏、双抽屉、编辑器与工作区框架
- 底部状态栏与切换按钮
- 设置、导入、发布、更新、素材库等对话框
- 菜单、搜索面板、输入框、按钮、骨架屏、图片编辑辅助控件
- 滚动条、焦点、悬停、禁用与状态反馈

## 非目标

- 跟随系统
- 多套暗色主题或强调色自定义
- 改变 Markdown 文章主题、代码主题或导出内容
- 为暗色模式重构现有组件结构

## 验证

以单元/源码契约测试覆盖模式合法化、持久化、根属性同步、状态栏位置和语义、CodeMirror 原位重配置、预览白色隔离及暗色 token。最后执行全量测试、生产构建、`git diff --check`，并在可用时进行亮/暗两态视觉检查。
