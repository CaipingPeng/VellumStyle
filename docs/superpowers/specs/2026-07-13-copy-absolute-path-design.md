# 文件树复制绝对路径设计

**日期：** 2026-07-13
**状态：** 已获用户口头确认

## 目标

为文档树中的文件和文件夹右键菜单增加“复制绝对路径”。点击后，将该条目在本机文档目录中的真实绝对路径写入系统剪贴板，并明确反馈成功或失败。

## 范围

### 包含

- 文件节点和文件夹节点均显示“复制绝对路径”。
- 使用 Tauri 后端解析相对路径并返回绝对路径。
- 复用现有纯文本剪贴板工具。
- 成功、失败和 Web 调试模式不可用时显示 Toast。
- 增加前端交互测试和 Rust 路径处理测试。

### 不包含

- 复制文档内容。
- 复制 `file://` URL。
- 为 Web 调试模式伪造本地路径。
- 引入新的剪贴板插件或 Rust 剪贴板依赖。
- 调整文档根目录或现有文件管理逻辑。

## 交互设计

现有节点右键菜单保持原有视觉样式，菜单顺序为：

1. 打开文件位置
2. 复制绝对路径

“复制绝对路径”使用复制图标。点击后立即关闭菜单并执行异步复制：

- 成功：Toast 显示“绝对路径已复制”。
- 后端解析、条目不存在或剪贴板写入失败：Toast 显示可理解的错误信息。
- Web 调试模式：Toast 提示“Web 调试模式无法复制本地绝对路径”。

系统剪贴板可能受权限、远程桌面策略或其他程序占用影响，因此保留成功与失败反馈，不能假设复制必然成功。

## 架构与数据流

1. `TreeNode.tsx` 增加 `onCopyAbsolutePath(path)` 回调属性和右键菜单按钮。
2. `DocTree.tsx` 将该回调连接到 `useDocActions`。
3. `useDocActions.ts` 调用前端文档工具获取绝对路径，再调用现有 `copyPlainText`：
   - 只有剪贴板工具返回 `true` 才显示成功 Toast。
   - 返回 `false` 或抛出异常均显示错误 Toast。
4. `src/utils/documents.ts` 增加 `getEntryAbsolutePath(path)`：
   - Tauri 模式调用新命令 `get_entry_absolute_path`。
   - Web 模式拒绝请求并返回明确错误。
5. `src-tauri/src/documents.rs` 增加 Tauri 命令：
   - 复用 `resolve_in_documents`，确保路径仍受文档目录边界约束。
   - 检查目标实际存在。
   - 返回平台原生格式的绝对路径字符串。
6. `src-tauri/src/lib.rs` 注册新命令。

## 安全与错误处理

- 前端仅传递文档树节点持有的相对路径，但后端仍不信任输入。
- 后端继续通过 `resolve_in_documents` 拒绝 `.`、`..` 等越界路径。
- 返回路径前检查条目存在，避免复制无效路径。
- 无法将系统路径转换成字符串时返回明确错误，而不是返回损坏文本。
- 剪贴板失败不报告成功。

- `TreeNode.test.tsx`：点击菜单项后使用节点的相对路径调用 `onCopyAbsolutePath`，同时菜单关闭。

  - 剪贴板返回 `false` 时按失败处理。
  - 获取路径抛错时按失败处理。

- 测试不存在条目返回“条目不存在”。

## 验收标准

- 任意文件或文件夹节点的右键菜单均含“复制绝对路径”。
- 成功操作后，剪贴板文本与该条目的平台原生绝对路径完全一致。
- 文件路径保留实际扩展名（Markdown 文件包含 `.md`）。
- 文件夹路径指向文件夹本身。
- 路径含空格或中文时不添加引号、不转义、不截断。
- 复制失败时不会显示成功提示。
- 不影响现有“打开文件位置”、重命名、删除和拖拽功能。
