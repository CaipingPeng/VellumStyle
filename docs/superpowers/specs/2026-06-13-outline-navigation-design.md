# 文章大纲导航设计

> 项目：文澜排版
> 日期：2026-06-13
> 范围：为当前 Markdown 文档新增可收缩的多级标题导航栏。

## 目标

长文排版时，用户可以通过当前文档的多级标题快速跳转查看内容位置。大纲导航和多文件管理一样是工作区侧栏，由顶部按钮独立展开或收起。

## 关键决策

- 大纲按钮放在顶部左侧，紧邻多文件管理按钮。
- 多文件管理继续使用 `PanelLeft` 图标；大纲使用层级列表语义的图标，避免两个按钮都像“侧栏开关”。
- 文档侧栏和大纲侧栏可独立展开；两者同时打开时，文档侧栏在最左，大纲侧栏在其右侧。
- 大纲来源为当前 Markdown 内容中的 `#` 到 `######` 标题。
- 点击大纲项滚动到预览侧对应标题位置，并利用现有双向同步滚动让编辑器跟随。
- 当前预览位置对应的大纲标题需要高亮，帮助用户判断当前位置。

## 架构

### 1. 大纲数据模型

新增一个纯函数模块解析 Markdown 标题，输出：

- `id`：稳定项 id，用于 React key。
- `level`：标题级别 1 到 6。
- `text`：去掉 Markdown 标记后的显示文本。
- `line`：标题所在源码行号，0-based，与现有 `data-line` 锚点一致。

解析应跳过围栏代码块，避免代码示例里的 `#` 被识别成标题。空标题或无标题文档返回空数组。

### 2. 状态

在 Zustand store 中新增：

- `outlineOpen: boolean`
- `toggleOutline: () => void`

它和 `sidebarOpen` 一样属于运行期 UI 状态，不写入持久化。

### 3. 大纲侧栏

新增 `OutlineNav` 组件：

- 宽度和文档树一致，保持工具型界面的紧凑密度。
- 顶部显示简短标题和标题数量。
- 大纲项按 `level` 缩进，使用按钮元素保证键盘可达。
- 当前项用 accent 底色和文字高亮。
- 无标题时显示空状态。

### 4. 跳转与同步

`Preview` 暴露新方法：

- `scrollToLine(line: number): void`
- `getActiveHeadingLine(): number | null`

`scrollToLine` 在预览滚动容器中查找 `data-line >= line` 的最近元素，滚动到它的位置。由于预览滚动会触发现有同步滚动，编辑器会跟随。

当前位置高亮由 `App` 监听预览滚动，根据滚动容器顶部位置和大纲项行号计算 active line，再传给 `OutlineNav`。

## 边界处理

- 无标题：显示空状态，不报错。
- 标题在代码块中：不进入大纲。
- 预览 DOM 尚未渲染或没有匹配 `data-line`：跳转静默失败。
- 大纲项文字过长：单行省略，保留 title。
- 大纲开关按钮：保留 `title`，使用不同图标和 active 状态。

## 测试

- 单测 Markdown 标题解析：多级标题、代码块内 `#`、空标题、强调/链接等常见标题文本。
- 类型检查和构建通过。
- 浏览器手测：展开/收起大纲、点击多级标题、滚动预览时高亮变化、两个侧栏同时打开时布局不重叠。
