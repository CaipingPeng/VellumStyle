# 工作区侧栏展开横向溢出修复设计

> 项目：文澜排版 VellumStyle
> 日期：2026-07-13
> 范围：修复文档管理或大纲侧栏展开动画期间，页面底部短暂出现横向滚动条并带动底部区域上抬的问题。

## 背景与问题表现

工作区的文档管理侧栏和大纲侧栏以 `width: 0 → auto` 做 140ms 展开动画。展开过程中，侧栏逐帧占用更多横向空间，编辑器/预览分栏容器的可用宽度也随之逐帧缩小。

当前 `WorkspaceSplit` 用 `ResizeObserver` 测量容器宽度，将结果写入 React 状态，再按该状态计算编辑器和预览的固定像素宽度。浏览器布局、`ResizeObserver` 回调和 React 重渲染不是同一个同步布局步骤，因此动画中的某一帧可能出现：外层容器已经变窄，两个面板仍使用上一帧的固定像素宽度。多出的宽度向上传播到文档根节点，形成窗口最底部的横向滚动条；滚动条占用垂直空间后，底部状态栏看起来短暂上抬。动画结束且测量状态追上实际宽度后，溢出和抖动自行消失。

受影响的核心位置：

- `src/App.tsx`：两个侧栏的宽度动画；
- `src/components/Workspace/WorkspaceSplit.tsx`：容器测量、固定像素面板宽度和分栏渲染；
- `src/components/Workspace/WorkspaceSplit.test.ts`：当前组件测试把固定像素宽度当作渲染契约。

## 目标

- 在任一侧栏展开或收起的整个动画期间，不产生页面级横向溢出。
- 消除由底部横向滚动条出现/消失引发的底部区域抖动。
- 保留侧栏现有展开动画和减少动态效果偏好。
- 保留编辑器/预览分栏的拖动、键盘调整、双击恢复默认值和比例持久化行为。
- 保留 8px 固定分隔柄，以及基于可用宽度的最小面板宽度约束和 ARIA 数值。

## 非目标

本次不包含：

- 移除或重做侧栏动画；
- 更改文档管理、大纲、编辑器或预览的视觉设计；
- 更改文档侧栏自身的可调宽逻辑；
- 重构工作区状态持久化模型；
- 通过全局 `overflow-x: hidden` 或 `clip` 掩盖其他潜在溢出。

## 方案：由 CSS 同步分配面板宽度

将编辑器和预览从“React 根据测量结果写入固定像素宽度”改为“CSS Flexbox 根据当前容器宽度同步分配剩余空间”。

布局结构保持为：

```text
WorkspaceSplit（当前实际宽度）
├─ 编辑器：flex-grow = displayRatio，flex-basis = 0，min-width = 0
├─ 分隔柄：固定 8px，不参与伸缩
└─ 预览：flex-grow = 1 - displayRatio，flex-basis = 0，min-width = 0
```

两个面板的增长系数之和恒为 1，分隔柄固定占用 8px，因此浏览器会在同一次布局计算中，把“容器当前宽度减去 8px”的空间按 `displayRatio : (1 - displayRatio)` 分给两个面板。侧栏动画每改变一次可用宽度，Flexbox 都会直接用该帧的真实宽度重新分配，不需要等待观察器回调或 React 状态更新。

具体渲染契约为编辑器使用 `flex: <displayRatio> 1 0`，预览使用 `flex: <1 - displayRatio> 1 0`。两个面板保留 `min-width: 0`，避免其内容的固有宽度反向撑大工作区；移除由 `containerWidth` 推导出的内联 `width`，也移除面板上的 `flex-none`。

## `ResizeObserver` 的保留职责

`ResizeObserver` 和 `containerWidth` 状态继续保留，但不再参与物理像素宽度渲染，只用于交互与可访问性计算：

- 调用 `getWorkspaceRatioBounds` 计算当前允许的最小/最大比例；
- 调用 `clampWorkspaceSplitRatio` 得到用于显示的安全比例；
- 为 separator 提供 `aria-valuemin`、`aria-valuemax` 和 `aria-valuenow`；
- 在指针拖动、键盘调整和恢复默认值时执行比例边界限制；
- 当空间不足以同时满足两个 280px 最小宽度时，沿用现有等分降级行为。

观察器回调晚一帧最多只会让比例边界或显示比例稍后收敛，不会再留下旧的固定像素总宽度，因此不能制造页面级横向溢出。

## 交互与数据流

1. 应用读取持久化的 `workspaceSplitRatio` 并传给 `WorkspaceSplit`。
2. `WorkspaceSplit` 清洗比例，并根据最近一次测得的容器宽度得到 `displayRatio`。
3. React 仅把比例作为两个面板的 Flexbox 权重输出；浏览器使用容器该帧的真实宽度同步完成物理宽度分配。
4. 侧栏动画改变工作区宽度时，面板随 CSS 布局同步收缩或增长。
5. 指针拖动继续根据当前 `getBoundingClientRect()` 计算比例并即时更新草稿比例；拖动结束后提交持久化值。
6. 键盘方向键、修饰键大步调整、Home 键和双击恢复默认比例继续按现有规则工作。

## 边界情况

- 文档侧栏、大纲侧栏分别展开，或在一个已展开时再展开另一个：核心分栏始终只分配当帧剩余空间。
- 侧栏收起：分栏同步增长，不依赖 `ResizeObserver` 才填满空间。
- 非法、非有限或超出持久化范围的比例：继续由现有清洗逻辑回退到默认值。
- 极端有效比例：先按当前宽度约束，再作为 Flexbox 权重使用。
- 工作区小于两个最小面板宽度之和加分隔柄时：继续降级为 50/50；`min-width: 0` 保证内容不会撑破容器。
- `prefers-reduced-motion`：侧栏会立即切换宽度，分栏仍在同一布局步骤适配。
- Windows 非整数缩放和亚像素宽度：浏览器负责 Flexbox 的余数分配，避免由 JavaScript 四舍五入形成总宽度误差。

## 防御性溢出策略

本方案不把 `overflow-x: clip` 作为核心修复，也不在 `html`、`body` 或工作区根节点增加全局隐藏。受控诊断显示，仅裁剪可消除页面滚动条，但内部仍存在旧宽度溢出；这会掩盖根因并可能截断合法内容。

如果实施后的真实 WebView 回归测试发现第三方内容存在与本问题无关的独立溢出，应单独定位到具体内容容器后再决定是否局部裁剪，不纳入本次修复。

## 测试策略

### 单元与组件测试

更新 `WorkspaceSplit.test.ts` 的渲染契约：

- 不再断言编辑器/预览得到 `500px`、`600px` 等固定 `width`；
- 断言两个面板以 `displayRatio` 和 `1 - displayRatio` 作为 flex 权重，`flex-basis: 0` 且允许收缩；
- 断言拖动时权重即时变化，结束时仍只提交一次最终比例；
- 保留 separator 的方向、标签、ARIA 最小/最大/当前值测试；
- 保留键盘小步/大步、Home 或双击复位、`pointercancel` 和卸载清理测试；
- 增加或调整测试，证明 `ResizeObserver` 更新只影响约束/ARIA，不再向面板写入固定像素宽度。

现有 `workspaceSplitLayout` 纯函数测试继续覆盖比例清洗、边界、指针换算和键盘步进。若像素宽度换算函数在实现后已无生产调用，可在实施阶段删除该函数及其专属测试，避免保留误导性的旧布局契约。

### 动画期回归验证

沿用本次复现问题所用的本地 headless Edge/CDP 诊断方式，在实现验证阶段运行，不为这次单点修复新增 E2E 测试框架。诊断在侧栏动画开始、进行中和结束时逐帧检查：

```js
document.documentElement.scrollWidth === document.documentElement.clientWidth
```

至少覆盖：

- 文档侧栏：关闭 → 打开；
- 大纲侧栏：关闭 → 打开；
- 文档侧栏已打开 → 再打开大纲；
- 大纲侧栏已打开 → 再打开文档；
- 相同组合的收起路径；
- 800px 最小窗口宽度与 1200px 默认窗口宽度。

### 手工桌面回归矩阵

在 Windows/WebView2 下以 100%、125%、150%、200% 系统缩放抽查上述侧栏组合，并验证：

- 动画期间最底部不出现横向滚动条，状态栏不抬升；
- 分隔柄拖动过程连续，结束后比例正确保存；
- 键盘调整、双击复位正常；
- 刷新或重启后恢复持久化比例；
- 0.2、0.8 等极端持久化比例会按当前空间正确约束；
- 编辑器、预览、文档树和大纲内容没有被意外裁切。

最后执行 `npm test` 和 `npm run build`。

## 验收标准

- 文档侧栏或大纲侧栏展开/收起的任意动画帧，`document.documentElement.scrollWidth` 不大于 `clientWidth`。
- 两个侧栏可以单独或连续展开，窗口底部均不出现横向滚动条，状态栏没有可见上抬。
- 编辑器宽度、预览宽度与 8px 分隔柄之和始终等于分栏容器当前宽度（允许浏览器亚像素精度误差）。
- 分栏拖动、键盘操作、双击复位、ARIA 值和持久化比例行为不回退。
- 在 800px/1200px 窗口与 Windows 100%/125%/150%/200% 缩放抽查中未复现。
- `npm test` 与 `npm run build` 通过。
