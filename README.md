本项目是用rust+tauri构建的桌面微信公众号排版工具。本工具排版仅适用于Markdown成稿的稿件，因此使用本软件的朋友一般需要具备一定的Markdown语法基础。当然，道友能看到本文，相信你已经来到了Github，故不会Markdown是我多虑了。

![文澜排版主界面](assets/hero.png)



## 一、软件覆盖的功能（跟随提交实时更新）

1. 基本功能：

   - 实时编辑与预览：左侧编辑Markdown，右侧根据**选定的主题**实时渲染排版效果。编辑器支持部分常用Markdown语法占位符的快速插入，不过目前只能手动触发，还没有设置快捷键。
   - 页面配色：提供亮色与暗色两种全局主题色，切换按钮在软件状态栏最右端。

2. ⭐”发布“功能⭐：

   - 复制到微信：软件将渲染后的html代码写入剪贴板，使得用户在公众号网页端文章编辑区粘贴后能够保留在软件内看到的样式效果。

   - 发布到草稿箱：用户可以直接在软件内，上传封面图，填写标题、作者，调整评论功能后，调用微信公众号草稿箱接口生成将文章放到微信公众号平台的草稿箱，非常方便。

     封面图片可以用两种方式确定：①从本地上传，图片会上传到公众号永久素材库，然后返回mediaID；②直接用自己公众号永久素材库选择，如果需要用你已经上传过的图片，那么这一种方式将会非常方便。

     ![复制与发布到公众号](assets/copy-and-publish.png)

3. ⭐图床与图片上传⭐：

   软件目前支持四种图片处理方式，①Ctrl+V粘贴；②通过快捷语法按钮上传本地图片；③通过快捷语法按钮**复用永久素材库的图片**；④导入Markdown文档时，自动解析并处理原始图片。不管是哪一种，最终图片都会经软件调用后端上传服务，上传到自己公众号的永久素材库。

   图片上传后，文章图片的地址会被替换为微信永久素材库的官方链接。此般，解决了图床跑路导致文章图片失效的问题。

4. ⭐文章管理与云同步⭐：

   - 文章管理：在本地用户目录，针对本软件会创建一个目录用于存档我们写过的文章。如下图左图所示，文章是树形管理的，也就是说你可以像在使用windows文件管理一样在软件内`新建`、`重命名`、`删除`、`拖动文件及文件夹`，也可以通过`右键菜单在系统文件管理器中打开文件位置`或者 直接复制`绝对路径`。删除非空的文件夹前会提示将被递归删除的子项数量，降低误操作风险。文档侧栏宽度可以自由拖动调整；打开深层文档时，软件会自动展开其所在目录。

      <img src="assets/documents-and-sync.png" alt="多文档管理与文件同步" style="zoom:50%;" />  

   - 云同步：为了满足多设备使用，软件使用坚果云免费的 WebDAV服务，把文件管理目录下的文件树同步到云端目录。在B设备上安装本软件后，配置好坚果云账号信息，即可恢复A设备的所有文章。

   - 本地优先：文章编辑和保存不依赖网络，云同步在本地保存完成后于后台异步执行，不会因为网络波动阻塞编辑。

5. ⭐主题系统⭐：

   软件内置 40+ 排版主题和 250+ Highlight.js/Base16 代码主题，支持搜索、分页、收藏和置顶。

   ![主题系统](assets/theme-gallery.png)

   - 自定义主题：除内置主题外，软件还支持导入主题模型 JSON 文件，或直接打开用户主题目录管理自定义主题。用户主题会与内置主题自动合并；同名用户主题可以覆盖内置主题，方便长期维护自己的排版方案。

6. ⭐应用自动更新⭐：软件启动的时候会自动检查新版本，也可以在“设置 → 关于”中进行手动检查。发现更新后可以查看版本号和更新内容，并在应用内直接下载安装；安装完成后软件会自动重启。这个功能极大提高了用户体验，版本更新无需用户再去下载安装包。

   <img src="assets/image-20260713131820259.png" alt="image-20260713131820259" style="zoom:50%;" />

7. ⭐ 批量 Markdown 导入与资源根目录⭐：

   - 批量导入：可以一次选择多个 Markdown 文件，并将其导入当前选中的文档目录。
   - 指定资源目录：有的markdown编辑器对语法进行了扩展，比如Obsidian使用 `！[[图片名字]]`这种语法，在自动尝试在子目录查找资源失败后，可以根据指定资源目录进行精准定位，然后解析上传。

8. 其他”也许“对你来说实用的小功能：

   - 自动保存与启动恢复：编辑内容会自动保存为本地的Markdown 文件，切换文档或关闭窗口前也会主动刷新尚未落盘的内容。软件会记住上次打开的文档，下次启动时自动恢复编辑现场。

   - Markdown文档导入：该功能区别于其他的编辑器逻辑——只上传文本，不做后处理，本软件上传过程中会单独处理所有的图片语法，不论你使用的是obisidian的"**![[]]**"语法，还是`<img src="xxx" alt="xxx">`的html标签语法，最终都会在图片上传后统一到`![imgDescription](imgUrl)`这样的Markdown图片插入标准语法。

     软件为了保留文档在其他编辑器编辑时留下的图片缩放等额外参数，对标准语法进行了轻度扩展，原始缩放会转换到`![imgDescription](imgUrl =缩放参数)`语法。

   - 扩展语法解析支持：①支持Mermaid代码直接渲染为SVG图形，并确保粘贴/发布到草稿箱后能保留SVG图形；②支持标准的 `$行内公式$`和`$$行间公式$$`数学公式渲染；③支持”参考文献“功能，文章如果使用了 `[xxx](url "title")`超链接语法，渲染为 $\lceil \text{xxx}\rfloor ^{[\text{文献序号}]}$这种样式，并自动在文末插入一条 $[\text{文献序号}]\space \text{title：url}$​​这样的文献。

   - 大纲导航：文档如果有标题语法，即 `# xxx`、 `## xxx`这种，左侧有可折叠导航栏，展开后可依据多级标题形成目录导航，方便长文定位。

   - 微信接口白名单助手：如下图右边分图，可以在设置中一键获取并复制当前的出口公网 IP，同时跳转到微信开发者平台的 IP 白名单配置页面。软件启动时也会检测出口 IP 是否发生变化，变化后展示新旧 IP 并主动提醒更新白名单，如下图左图所示，减少因为动态公网 IP 导致的接口调用失败。

     ![image-20260713131638646](assets/image-20260713131638646.png)

   - 同步滚动：编辑器和预览器实现了双向高精度同步滚动。

   - 搜索与替换：编辑器支持通过 Ctrl+H 打开中文搜索替换面板，可以查找、逐项替换或全部替换当前文档内容。

     <img src="assets/image-20260713151014924.png" alt="image-20260713151014924" style="zoom: 67%;" />

   - 比例调整：编辑器与渲染器的比例可调整，并可以在双击分隔条后恢复55开比例。

   - 可视化图片缩放：在预览区选中图片后，可以拖动四角缩放手柄调整图片宽度。缩放过程保持原始比例并实时显示宽度百分比，调整的结果会自动写回 Markdown 的图片尺寸扩展语法。

   - 可视化样式编辑：可以直接在预览区单击选中标题、段落、列表、引用、图片等文章元素，在样式面板中调整字号、颜色、背景、行高、字间距、对齐方式、间距、边框和圆角等属性。样式修改分为“临时修改”和“永久修改”两种模式，临时修改只影响当前预览；永久修改会写入用户主题文件，并在后续文档及下次启动时继续生效。

   -  多终端预览：预览区支持“放开展示”“微信桌面端渲染”和“手机端渲染”三种宽度模式，可以在发布前检查文章在桌面端与移动端的实际排版效果。

   - 文档导出：支持 PNG 长图、独立 HTML、A4 PDF、Markdown 原文导出。虽然但是，感觉这个功能没啥用。

## 二、下载与安装

仓库已经发布了正式版本，第一次需要从 [GitHub Releases](https://github.com/CaipingPeng/VellumStyle/releases/latest) 下载系统对应的安装包。

当前 Tauri 配置主要面向 Windows：

| 平台 | 当前状态 |
| --- | --- |
| Windows | 已配置 `msi` 和 `nsis` 打包目标 |
| macOS | 作者没有环境，故尚未构建，期待你来贡献 |
| Linux | 尚未构建 |

软件安装后，详细的配置见[VellumStyle-文澜排版帮助文档](https://my.feishu.cn/docx/RUDpd1zWnoWuuyx0uFxcahIGnmC)

## 三、开源贡献与二次开发

>  欢迎提交 Issue、主题样式、文档改进和 PR。

环境要求：

- Node.js 20 或更高版本
- npm 10 或更高版本
- Rust 1.77.2 或更高版本
- Windows 桌面构建需要 WebView2 Runtime 和 Microsoft C++ Build Tools
- macOS / Linux 需要安装 [Tauri v2 对应的系统依赖](https://v2.tauri.app/start/prerequisites/)

安装依赖并启动完整桌面开发环境：

```bash
npm install
npm run tauri
```

如果只调试编辑器、预览和主题等前端功能，可以运行：

```bash
npm run dev
```

Web 模式不包含 Tauri 后端，因此文件选择、图片上传、草稿箱发布、本地文档树、PDF 导出和坚果云同步等功能不可用。

提交前请完成基础检查：

```bash
npm test
npm run build
cargo test --manifest-path src-tauri/Cargo.toml
```

提交问题时请说明操作系统、复现步骤、预期结果和相关报错。日志或截图中的 AppSecret、坚果云授权密码及未发布文章内容请先脱敏。
